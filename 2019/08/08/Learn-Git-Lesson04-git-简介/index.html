.git简介

==============

知识点

  • 了解 .git 目录
  • HEAD
  • config
  • refs
  • objects

.git 目录

  • HEAD ： 标识了当前指向的分支
  • config ： 本地的配置信息

HEAD

修改分支，观察HEAD内容的变化

config

修改config文件的用户名，观察变化

最后再将用户名改回

refs

  • heads
  • tags
  • remote
heads

heads 中存放着所有的分支,可以把分支看作独立的工作空间，在一定时间内保持各自研发的独立性，等分支开发的模块成熟后再合体到集成分支。

heads 下有两个文件：master and temp

tags

Tag存储的是一个里程碑，里面指向了一个commit

objects

这两位+进去后的哈希值合在一起就是一棵树（Tree）

继续进入这棵树

任何文件的内容相同，其实就是同一个blob
